Create data/fetch_smiles.py following the Phase 1 script pattern (argparse, logging, tqdm, skip-if-exists, summary table).

Module-level constants:
```python
PUBCHEM_URL = "https://pubchem.ncbi.nlm.nih.gov/rest/pug/compound/cid/{cid}/property/CanonicalSMILES/txt"
SEMAPHORE_LIMIT = 5  # ≤5 req/sec per PubChem policy
CACHE_PATH = Path("data/raw/pubchem_cache.json")
INGREDIENTS_CSV = Path("data/raw/ingredients.csv")
MOLECULES_CSV = Path("data/raw/molecules.csv")
```

Public function `fetch_smiles(force: bool = False) -> dict`:
1. **Skip-if-exists gate**: If CACHE_PATH exists and not force, load and verify completeness. If set(cache.keys()) == set(all_pubchem_ids), log "[SKIP] pubchem_cache.json already complete ({n} entries)" and return cache. Use string keys in JSON (json.dump converts int keys to strings — compare str(id)).
2. **Extract FlavorDB2 SMILES**: Read ingredients.csv, parse molecules_json column (json.loads per row), build {pubchem_id: smile} dict. Skip rows where smile field is empty string or None — those become gap candidates.
3. **Find gaps**: Load molecules.csv, collect all pubchem_ids. Identify IDs missing from FlavorDB2 extraction.
4. **PubChem async fetch** (only for gap IDs): Use asyncio.run(fetch_missing_smiles(gap_ids)) with the async pattern from RESEARCH.md. Distinguish 200/404/5xx explicitly — do NOT store 5xx as null.
5. **Merge and write**: Merge FlavorDB2 dict + PubChem dict. For every pubchem_id in molecules.csv, ensure an entry exists (even if value is None). Write with json.dump, indent=2.
6. **Gate check**: Assert set(str(k) for k in cache.keys()) == set(str(id) for id in all_pubchem_ids). Log error and raise if not satisfied.
7. **Summary table**: Log totals: total={n}, with_smiles={n}, missing_smiles={n} (null entries).

Async helper functions (module-level, not nested):
- `async def fetch_smiles_for_id(client, sem, pubchem_id)` → returns (pubchem_id, smiles_or_none)
- `async def fetch_missing_smiles(missing_ids)` → returns dict

CLI: `if __name__ == "__main__": main()` with argparse --force flag, same pattern as data/scrape_flavordb.py.

Logging: append to logs/pipeline.log (same shared log), tqdm-compatible console handler. Logger name: "fetch_smiles".

CRITICAL: The molecules_json column in ingredients.csv is a per-ingredient list of molecule dicts. Multiple ingredient rows may share a pubchem_id (same molecule in multiple ingredients). Deduplicate by pubchem_id when building the FlavorDB2 extraction dict.
