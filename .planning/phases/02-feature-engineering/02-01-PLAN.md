Test functions to create (exact names required — Plans 02-04 reference these):

FEAT-01 (fetch_smiles.py):
- test_smiles_cache_coverage: Asserts pubchem_cache.json exists, loads as JSON dict, verifies set(cache.keys()) covers all pubchem_ids from data/raw/molecules.csv. Marks xfail if file not found.
- test_smiles_missing_logged: Creates a mock pubchem_id that should be absent (e.g., id=0), verifies the fetch function stores None for 404 rather than omitting the key. Tests the "null entry = legitimate miss" contract.

FEAT-02 (build_features.py):
- test_rdkit_descriptors: Calls compute_molecule_features(pubchem_id=702, smiles="CCO", common_name="Ethanol"), asserts returned dict contains keys MW, logP, HBD, HBA, rotatable_bonds, TPSA with non-None float values.
- test_rdkit_sanitization_logged: Calls compute_molecule_features(pubchem_id=1, smiles="INVALID_SMILES", common_name="bad"), asserts returned dict has all descriptor columns as None and morgan_fp_bytes as None. Does NOT assert log output (avoids log capture complexity).

FEAT-03:
- test_morgan_fingerprint: Calls compute_molecule_features(pubchem_id=702, smiles="CCO", common_name="Ethanol"), asserts morgan_fp_bytes is bytes of length 1024.

FEAT-04:
- test_tanimoto_edges: Creates two known similar SMILES (CCO and CCCO, ethanol + propanol), builds ExplicitBitVects via AllChem.GetMorganFingerprintAsBitVect, calls compute_tanimoto_edges(fps_and_ids). Asserts result is a list; does NOT assert threshold (ethanol+propanol may be < 0.7 — test existence, not values). For threshold: use two near-identical SMILES like "CC(C)O" and "CC(C)O" (same molecule = sim 1.0) to assert at least one edge exists.

FEAT-05:
- test_texture_encoding: Calls encode_texture("herb"), asserts result is a list of length 5 with exactly one 1 and four 0s (one-hot constraint). Also calls with "unknown_xyz" and asserts result is still length-5 (graceful fallback).

FEAT-06:
- test_temperature_encoding: Calls encode_temperature("herb"), asserts result is a list of length 4 with exactly one 1 and three 0s. Also tests "unknown_xyz" returns length-4 fallback.

FEAT-07:
- test_cultural_context: Calls build_cultural_context_vectors(recipes_df) with a tiny in-memory DataFrame (2 recipes: "Indian Tikka Masala" with ingredient "chicken", "Mexican Tamales" with ingredient "corn"). Asserts the returned dict has entries for "chicken" and "corn", each a list of length 10, with at least one non-zero value.

FEAT-08:
- test_flavor_profile_vocab: Creates a tiny mock molecules DataFrame with flavor_profile column containing "sweet@floral" and "sweet@bitter@umami". Calls build_flavor_vocab(mol_df) and encode_flavor_profile(mol_rows, vocab_index). Asserts: vocab contains "sweet", "floral", "bitter", "umami" (4 items sorted); encoded vector has length matching vocab; "sweet" dimension is 1 in the result.

FEAT-09 (integration):
- test_parquet_outputs_exist: Asserts all four parquet files exist: data/processed/ingredients.parquet, data/processed/molecules.parquet, data/processed/tanimoto_edges.parquet, data/processed/cooccurrence.parquet. Marks xfail if any are missing.
- test_molecules_parquet_schema: Loads data/processed/molecules.parquet, asserts columns include: pubchem_id, smiles, MW, logP, HBD, HBA, rotatable_bonds, TPSA, morgan_fp_bytes. Marks xfail if file not found.
- test_ingredients_parquet_schema: Loads data/processed/ingredients.parquet, asserts columns include: ingredient_id, name, category, texture_0, temperature_0, cultural_context_0, flavor_profile_0. (Prefixed column names — the exact suffix numbering is implementation detail, but prefix must be present.) Marks xfail if file not found.
