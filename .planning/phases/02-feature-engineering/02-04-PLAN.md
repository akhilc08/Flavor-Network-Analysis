TEXTURE_MAP dict (Claude's discretion for exact mappings):
Map FlavorDB2 ingredient categories to one of: "crispy", "soft", "creamy", "chewy", "crunchy".
Reasonable defaults:
- crispy: nut, seed, snack
- soft: fruit, vegetable, mushroom
- creamy: dairy, fat, oil
- chewy: meat, seafood, grain
- crunchy: vegetable (raw), cereal, cracker
- Default (unknown category): soft

TEMPERATURE_MAP dict:
- raw: herb, spice, condiment, vegetable, fruit, nut, seed
- cold: dairy, beverage (cold), dessert (ice cream)
- warm: grain, bread, legume, pulse
- hot: meat, seafood, soup, sauce
- Default: warm

CUISINE_CATEGORIES constant (ordered list, defines column order):
["Italian", "Asian", "Mexican", "French", "American", "Indian", "Mediterranean", "Middle Eastern", "Japanese", "Thai"]

CATEGORY_KEYWORDS dict (Claude's discretion — use RESEARCH.md pattern):
Keyword lists for each cuisine. Apply case-insensitive match to recipe_name. First-match wins.
Log at startup: "Loaded CATEGORY_KEYWORDS with {n} cuisines"
Log at build time: "AllRecipes category assignment: {n} matched, {n} unmatched ({pct}%)"
WARN if unmatched > 10% of total.

Public functions to add:

`encode_texture(category: str, moisture_content: float | None = None) -> list[int]`:
- Look up category in TEXTURE_MAP (case-insensitive, strip whitespace)
- If moisture_content is non-null and < 10.0 (very dry): override to "crispy"
- If moisture_content is non-null and > 80.0 (very moist): override to "soft"
- Return 5-dim one-hot over ["crispy", "soft", "creamy", "chewy", "crunchy"]
- Unknown category: return [0, 1, 0, 0, 0] (default "soft")

`encode_temperature(category: str) -> list[int]`:
- Look up category in TEMPERATURE_MAP (case-insensitive)
- Return 4-dim one-hot over ["raw", "cold", "warm", "hot"]
- Unknown category: return [0, 0, 1, 0] (default "warm")

`build_cultural_context_vectors(recipes_allrecipes_path: Path) -> dict[str, list[int]]`:
- Read recipes_allrecipes.csv (76 rows, columns: recipe_name, ingredients)
- For each recipe, assign cuisine category via CATEGORY_KEYWORDS (first match, case-insensitive)
- Parse ingredients column (comma-separated) into individual ingredient names
- For each ingredient in recipe: increment that cuisine dimension
- Return dict: {ingredient_name_lower: [count_Italian, count_Asian, ..., count_Thai]}
- For any dimension with count > 0, the final vector for Phase 3 will be binarized to 1
- Return 0-vectors (length 10) for ingredients that appear in no matched recipes

`build_flavor_vocab(molecules_df: pd.DataFrame) -> tuple[list[str], dict[str, int]]`:
- Parse flavor_profile column (@ delimiter) over all rows
- Build sorted vocabulary (sorted() for determinism)
- Return (FLAVOR_VOCAB list, FLAVOR_VOCAB_INDEX dict)

`encode_flavor_profile(molecule_rows: list[dict], vocab_index: dict[str, int]) -> list[int]`:
- Union of flavor_profile tags across all molecules belonging to one ingredient
- Return multi-hot vector of length len(vocab_index)
- Unknown tags: silently skipped (vocab is fixed at dataset level)
