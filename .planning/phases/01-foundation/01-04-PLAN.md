RapidFuzz join pattern:
```python
from rapidfuzz import process, fuzz

def fuzzy_join(flavordb_names, foodb_name, threshold=85):
    match, score, _ = process.extractOne(
        foodb_name,
        flavordb_names,
        scorer=fuzz.token_sort_ratio
    )
    if score >= threshold:
        return match, score
    return None, score
```

CRITICAL: Use fuzz.token_sort_ratio, NOT fuzz.ratio
- fuzz.ratio breaks on "garlic, roasted" vs "roasted garlic" (word order sensitive)
- token_sort_ratio normalizes token order before comparing (correct for food names)

FooDB download note (RESEARCH.md pitfall):
- FooDB is a 952MB tar.gz manual download from foodb.ca/downloads (CC BY-NC 4.0)
- The pipeline script MUST check if data/raw/foodb/ exists and print clear instructions if not
- Do NOT attempt programmatic download (license compliance + size)
- Check for: data/raw/foodb/Food.csv and data/raw/foodb/Compound.csv (or similar)
  (Exact column names are unknown until first extraction — handle gracefully)

FooDB missing data handling (from CONTEXT.md locked decision):
- If FooDB fuzzy match < 300 ingredients: print prominent WARNING, continue (do NOT crash)
- Never threshold-lower automatically
- Example: logger.warning("WARNING: FooDB join matched only N/M ingredients (threshold=85). Expected >= 300.")

Idempotent join: check if molecules.csv already has "foodb_matched" column; if yes, skip unless --force
